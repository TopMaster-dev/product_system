ダッシュボード

トップ画面です。カードをクリックすると各画面へ移動します。

  • 未解決アラート（赤）: 対応表が無くて止まっている注文の件数。0 以外なら対応が必要です。
  • 同期エラー（要対応）（赤）: 在庫反映に失敗した件数。0 以外なら「同期エラー」画面で確認します。
  • 承認待ちの照合（黄）: リコンサイルで承認待ちの件数。0 以外なら「リコンサイル」画面で確認します。
毎日の確認ポイント: この3つが 0 になっていれば、基本的に運用は正常です。

在庫一覧

SKUごとの現在の在庫数を確認します。

検索・絞り込み

  • 検索ボックスに SKUコード または 商品名 の一部を入れて {{ ui('検索') }}。
  • 「在庫状態」で すべて / マイナス / ゼロ / 低在庫（1〜9）/ 正常（10以上）/ 売れ筋 に絞り込めます。
  • 画面上部の件数バッジ（マイナス◯ / ゼロ◯ / 低在庫◯）は、クリックするとその状態だけに絞り込めます。

表示・並び替え・出力

  • 「状態」列にラベル（{{ pill('neg','マイナス') }} {{ pill('zero','ゼロ') }} {{ pill('low','低在庫') }} {{ pill('ok','正常') }}）が表示されます。初期表示は問題のあるSKUが上（マイナス→ゼロ→低在庫→正常）に並びます。
  • 列の見出し（SKU / 商品名 / 在庫数 / 状態 / 最終更新）をクリックすると並び替えできます。
  • 右上の {{ ui('CSV出力') }} で、いま表示している条件のまま一覧をダウンロードできます。
  • 直近の販売が多いSKUには {{ ui('売れ筋') }} バッジが付きます。売れ筋かつ在庫が少ない場合は「売れ筋商品が在庫不足です」と警告が出ます（補充の目安）。
  • 各行の {{ ui('履歴 →') }} から、その SKU の在庫の動きを確認できます。
ここでは在庫数の確認のみです。数を変えるときは「手動調整」または「リコンサイル」を使います。

リコンサイル（在庫の突き合わせ）／ 在庫CSV取込

CROSS MALL の在庫CSVとシステムの基準在庫を突き合わせ、ズレ（差分）を承認して補正します。毎日の在庫合わせの中心機能です。

3-1. CSVを取り込む

アップロード→ 検証→ 確認→ 実行
  1. 「リコンサイル」→ 右上の {{ ui('在庫CSVを取込') }} をクリック。
  2. アップロード: CROSS MALL の在庫CSV（Shift-JIS のまま）を選び {{ ui('検証して確認へ') }}。必須列は 商品コード と 在庫数量。
  3. 確認: 取り込む内容が表示されます。この時点では在庫は変わりません。「現在庫 → CSV在庫」と差分が一覧で見え、CSVに問題（文字コード違い・列不足・数値でない行）があれば警告が出ます。
  4. 問題なければ {{ ui('この内容で実行') }} をクリック。「照合」が作成され差分確認画面へ移動します。

3-2. 差分を承認して反映する

  1. 差分ごとに {{ ui('承認') }}（CSVの在庫数に補正）または {{ ui('スキップ') }}（今回は変えない）を選びます。
  2. すべて処理すると {{ ui('確定する') }} が押せます。
  3. {{ ui('確定する') }} で照合が完了。チャネル（楽天・Shopify）への反映は、この後の定期同期で自動的に行われます。

3-3. その他

  • 承認前の照合は {{ ui('取消') }} できます。各照合は {{ ui('CSV出力') }} で内容をダウンロードできます。
  • リコンサイルは毎朝自動でも実行されます（承認待ちの状態で作成）。ダッシュボードの「承認待ちの照合」を見て承認・確定してください。
{{ num('4') }}

同期エラー

楽天・Shopify への在庫反映が失敗したときに記録される画面です。

  1. 一覧に、失敗した SKU・チャネル・日本語の原因と対処が表示されます（例:「チャネルAPIのレート制限に達しました。数分待ってから再実行してください。」）。
  2. 原因を確認し、右側の {{ ui('再実行') }} を押すと、最新の在庫数で再度反映します（成功=緑 / 再失敗=赤 のメッセージ）。
  3. 上部のフィルタで、状態・チャネル・SKU で絞り込めます。「詳細データ」で送信内容・エラー詳細も確認できます。
対処のコツ: 一時的なエラー（タイムアウト・混雑）は、少し待って {{ ui('再実行') }} で解決することがほとんどです。何度も失敗する場合は技術担当へご連絡ください。

マッピング（SKU対応表）

チャネルのSKU（楽天・Shopify・CROSS MALL）と社内SKUの対応表を管理します。

  • 検索・チャネルで絞り込み。
  • {{ ui('マッピング追加') }} で1件ずつ登録、または画面内の取込からCSVで一括登録。
  • {{ ui('CSV出力') }} で現在の対応表をダウンロードできます。
通常は初期設定済みです。新商品追加時などに使います。
{{ num('6') }}

アラート（未マッピング注文の解決）

対応表が見つからない注文が来ると、ここに表示されます。上部は 未対応 / 対応中 / 解決済み の3タブに分かれ、各タブに件数バッジが付きます。「未解決件数」「直近7日の新規」「チャネル別件数」のサマリーも表示されます。

各行には、注文された 商品名 と 商品管理番号 が表示され、どの商品かを確認できます（楽天の場合は楽天の商品名）。解決時の社内SKU選択にも商品名が表示されるので、どのSKUに紐付けるか選びやすくなっています。

対応の流れ

  1. 未対応タブの行で、次のいずれか。
    • すぐ解決 … 正しい社内SKUを選んで {{ ui('解決') }}。対応表が登録され、保留中の注文が自動で再処理されます（在庫も反映）。
    • あとで対応 … 担当者名（任意）を入れて {{ ui('対応中にする') }}。行が「対応中」タブへ移動し、誰が対応中かが記録されます。
  2. 対応中タブで、担当者が確認のうえ {{ ui('解決') }} して完了。
  3. 解決済みタブで過去の解決を確認。チャネル / チャネルSKU（部分一致）で絞り込みも可能です。
ダッシュボードの「未解決アラート」が赤いときは、この画面で解決してください。

手動調整

棚卸・破損・検品差異などで在庫を手で増減します。確認画面を挟むので誤操作を防げます。

  1. SKU を選択（現在の在庫数がカッコ内に表示）。
  2. 調整数を入力（正の数で増加、負の数で減少。例: +5 / -3）。
  3. 理由を入力。よく使う理由は理由テンプレート（不良品 / 破損 / 検品NG / 紛失 / 棚卸差異 / POPUP戻り在庫）から選ぶと自動入力されます（自由入力も可）。
  4. 入力すると 「現在 ◯ → 変更後 ◯」 のプレビューがその場で表示されます。
  5. {{ ui('確認画面へ') }} → 内容を再確認し {{ ui('確定して適用') }} で反映（{{ ui('戻る') }} で修正も可）。調整後にマイナスになる場合は拒否されます。
  6. 画面右の 「最近の手動調整」 に、適用した調整がすぐ反映されます。
すべての調整は履歴（イベントログ）に、理由と操作者付きで自動記録されます。

イベントログ

在庫が動いたすべての履歴を検索できます（注文・キャンセル・手動調整・棚卸・入荷）。

  • SKU・種別・チャネル・期間で絞り込み。
  • 「なぜこの在庫数なのか」を後から追跡するときに使います。
